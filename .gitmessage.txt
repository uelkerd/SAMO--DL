# SAMO-DL Commit Message Template
#
# Format: <type>(<scope>): <subject>
#
# Types:
#   feat:     A new feature
#   fix:      A bug fix
#   docs:     Documentation changes
#   style:    Code style changes (formatting, etc.)
#   refactor: Code refactoring
#   test:     Adding or updating tests
#   chore:    Maintenance tasks
#
# Rules:
#   - ONE purpose per commit (no "and", "also", "plus")
#   - Subject line < 50 characters
#   - Use imperative mood ("Add" not "Added")
#   - No period at end of subject line
#
# Examples:
#   feat(auth): add user authentication system
#   fix(models): resolve memory leak in model loading
#   chore(deps): update dependency versions
#   refactor(limiter): simplify rate limiter logic
#   docs(api): update API documentation
#   test(validation): add unit tests for validation functions
